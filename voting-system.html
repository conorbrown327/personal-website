Voting System

A semester-long team project that explored both the waterfall and the agile scrum design processes. The first portion of this project used the waterfall design process to implement the system. My team and I were tasked with creating a system that could tally votes and determine a winner for both Instant Runoff and Open Party Listing elections. This system would do this by reading in a .csv file from the command line that contained a list of ballots cast by voters. Our system processed this information and determined a winner for that election. It outputs a quick summary to the command line as well as an audit file that shows every step of our system and a media file that provides a more detailed overview of the election results.

For the second half of the project we followed the agile scrum model to add new features to the existing system. The new features included the ability to input multiple files for one election simultaneously, adding a new popularity only election type and more. My team's sprint backlogs provide information on all features added during the two week sprint.

View this project on Github. My team's SRS and SDD from the waterfall portion of the project contain a lot of great information on the overall system.
