Drone Delivery Simulation

A semester long team project that encompassed many of the techniques used in modern day software engineering. This included testing, github, UMLs, factory patterns, strategy patterns and observer patterns. The project itself was a package delivery simulation that would schedule drones and later robots to deliver packages to customers. The project was divided into three iterations discussed below.

  • Iteration 1: The main goal of Iteration 1 was to get the simulation working with a single delivery. This iteration included a UML, initial testing, and an analysis on the concrete factory pattern used to create entities within the simulation within the Doxygen mainpage.
  • Iteration 2: Included creating a robot entity that deliverd packages via ground only, adding an observer pattern that made alerts when events within the event happened, adding a strategy pattern to delivery entities that would select between three route options, smart, beeline and parabolic. Finally, our team added battery levels to drones and robot that would drain over time.
  • Iteration 3: For Iteration 3 our team implemented a battery level indicator to the drones and robots. The color of the indicator changed at specific battery levels to indicate how much charge remained.
